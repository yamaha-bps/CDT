3 # CDT: Constrained Delaunay Triangulation
4 
5 Numerically robust C++ implementation of constrained Delaunay triangulation (CDT)
6 - uses robust geometric predicates for numerical robustness
7 - can be consumed as header-only (default) or compiled (if `CDT_USE_AS_COMPILED_LIBRARY` is defined)
8 - permissively-licensed (MPL-2.0)
9 - backwards-compatible with C++03
10 - cross-platform: tested on Windows and Linux

28 ## <a name="algorithm"/>Algorithm</a>
29 
30 Implementation closely follows incremental construction algorithm by Anglada [[1](#1)]. During the legalization, the cases
31 when at least one vertex belongs to super-triangle are resolved using an approach as described in Žalik et. al [[2](#2)].
32 For efficient search of a triangle that contains inserted point randomized walking search is applied [[3](#3)]. To find the starting triangle we first find the nearest point using boost::rtree or using a closest random point.
33 
34 **Pre-conditions:**
35 - No duplicated points (use provided functions for removing duplicate points and re-mapping edges)
36 - No two constraint edges intersect each other
37 
38 **Post-conditions:**
39 - Triangles have counter-clockwise (CCW) winding

41 ## <a name="installation"/>Installation</a>
42 **Adding to CMake project directly**
43 
44 Can be done with [`add_subdirectory`](https://cmake.org/cmake/help/latest/command/add_subdirectory.html) command (e.g., see CDT visualizer's CMakeLists.txt).
45 ```Cmake
46 # add CDT as subdirectory to CMake project
47 add_subdirectory(../CDT CDT)
48 ```
49 **Adding to non-CMake project directly**
50 
51 To use as **header-only** copy headers from `CDT/include`
52 
53 To use as a **compiled library** define `CDT_USE_AS_COMPILED_LIBRARY` and compile `CDT.cpp`
54 
55 **Consume pre-build CDT in CMake project with [`find_package`](https://cmake.org/cmake/help/latest/command/find_package.html)**
56 
57 CDT provides package config files that can be included by other projects to find and use it.
58 
59 ```bash
60 # from CDT folder
61 mkdir build && cd build
62 # configure with desired CMake flags
63 cmake -DCDT_USE_AS_COMPILED_LIBRARY=ON -DCDT_USE_BOOST=ON ..
64 # build and install
65 cmake --build . && cmake --install .
66 ```
67 
68 ```CMake
69 # In consuming CMakeLists.txt
70 find_package(CDT REQUIRED CONFIG)
71 ```

73 ## <a name="details"/>Details</a>
74 
75 - Supports three ways of removing outer triangles:
76  - `eraseSuperTriangle`: produce a convex-hull
77  - `eraseOuterTriangles`: remove all outer triangles until a boundary defined by constraint edges
78  - `eraseOuterTrianglesAndHoles`: remove outer triangles and automatically detected holes. Starts from super-triangle and traverses triangles until outer boundary. Triangles outside outer boundary will be removed. Then traversal continues until next boundary. Triangles between two boundaries will be kept. Traversal to next boundary continues (this time removing triangles). Stops when all triangles are traversed.
79 
80 - Removing duplicate points and re-mapping constraint edges can be done using functions: `RemoveDuplicatesAndRemapEdges, RemoveDuplicates, RemapEdges`
81 
82 - Uses William C. Lenthe's implementation of robust orientation and in-circle geometric predicates: https://github.com/wlenthe/GeometricPredicates.
83 
84 - Boost is an optional dependency used for:
85  * **Performance:** rtree implementation for finding the closest point during points insertion: `nearestVertexRtree` is a faster than `nearestVertexRand`. Also `boost::container::flat_set` is used for faster triangle walking
86  * **Fall back** for standard library features missing in C++98 compilers.
87 
88  To opt in define `CDT_USE_BOOST` either in CMake or in a preprocessor.
89 
90 - A demonstrator tool is included: requires Qt for GUI. When running demo-tool **make sure** that working directory contains folder `test files`.

97 namespace CDT
98 {
99 
100 struct FindingClosestPoint

102  enum Enum
103  {
104 #ifdef CDT_USE_BOOST
105  BoostRTree,
106 #endif
107  ClosestRandom,
108  };
109 };

111 template <typename T>
112 class Triangulation
113 {
114 public:
115  /*____ Data ____*/
116  std::vector<Vertex<T> > vertices;
117  std::vector<Triangle> triangles;
118  EdgeUSet fixedEdges;
119  /*____ API _____*/
120  Triangulation(
121  const FindingClosestPoint::Enum closestPtMode,
122  const size_t nRandSamples = 10);
123  void insertVertices(const std::vector<V2d<T> >& vertices);
124  void insertEdges(const std::vector<Edge>& edges);
125  void eraseSuperTriangle();
126  void eraseOuterTriangles();
127  void eraseOuterTrianglesAndHoles();
128  // ...
129 }
130 
131 struct DuplicatesInfo
132 {
133  std::vector<std::size_t> mapping; ///< vertex index mapping
134  std::vector<std::size_t> duplicates; ///< duplicates' indices
135 };

138 DuplicatesInfo RemoveDuplicates(std::vector<V2d<T> >& vertices);

143 
144 template <typename T>
145 DuplicatesInfo RemoveDuplicatesAndRemapEdges(
146  std::vector<V2d<T> >& vertices,
147  std::vector<Edge>& edges);
148 
149 template <typename T>
150 std::vector<unsigned short> CalculateTriangleDepths(
151  const std::vector<Vertex<T> >& vertices,
152  const TriangleVec& triangles,

162 
163 } // namespace CDT

166 **Triangulated convex-hull example**
167 
168 ```c++
169 #include "CDT.h"
170 using Triangulation = CDT::Triangulation<float>;
171 
172 Triangulation cdt =
173  Triangulation(CDT::FindingClosestPoint::BoostRTree);
174 /*
175  // Without boost::rtree:
176  Triangulation(CDT::FindingClosestPoint::ClosestRandom, 10);
177 */
178 cdt.insertVertices(/* points */);
179 cdt.eraseSuperTriangle();
180 /* ... */ = cdt.vertices;
181 /* ... */ = cdt.edges;
182 ```
183 **Triangulated region constrained by boundary example**
184 
185 ```c++
186 // ... same as above
187 cdt.insertVertices(/* points */);
188 cdt.insertEdges(/* boundary edges */);
189 cdt.eraseOuterTriangles();
190 /* ... */ = cdt.vertices;
191 /* ... */ = cdt.edges;
192 ```
